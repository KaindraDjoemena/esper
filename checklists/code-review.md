# Code Review Checklist

## Purpose

Ensure code reviews are thorough and produce high-quality, actionable reports.

## When to Use

- peer code review
- code audits

## Code Verification

- [ ] Logic is correct
- [ ] Edge cases considered
- [ ] Error handling is appropriate
- [ ] Failure paths are handled
- [ ] Inputs validated
- [ ] Secrets protected
- [ ] No obvious unnecessary work
- [ ] Consistent with project conventions
- [ ] Appropriate abstraction level
- [ ] Missing test coverage identified
