# Security Review

## Purpose

Identify security vulnerabilities, evaluate attack surfaces, and recommend mitigations.

## When to Use

- security audits
- pre-release reviews
- threat modeling

## Deliverable Structure

Follow the canonical Report Structure.

Include the following security-specific sections:

### Attack Surface

Identify exposed interfaces, trust boundaries, and data flows.

### Positive Security Practices

Recognize good security decisions (e.g., defense in depth, input validation).
