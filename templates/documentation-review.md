# Documentation Review

## Purpose

Evaluate the accuracy, completeness, and clarity of project documentation.

## When to Use

- documentation audits
- reviewing READMEs or guides

## Deliverable Structure

Follow the canonical Report Structure.

Include the following documentation-specific sections:

### Accuracy

Does the documentation match the implementation?

### Missing Information

Identify important omissions or undocumented features.

### Outdated Information

Highlight obsolete content that needs removal or updating.
