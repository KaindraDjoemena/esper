# Performance Analysis

## Purpose

Identify performance bottlenecks, document findings with evidence, and propose optimized solutions.

## When to Use

- profiling code
- investigating latency or throughput issues
- performance optimization

## Deliverable Structure

Follow the canonical Report Structure.

Include the following performance-specific sections:

### Bottlenecks

List identified performance issues (e.g., algorithmic complexity, resource contention).

### Expected Impact

Estimate potential gains for the recommended improvements.
