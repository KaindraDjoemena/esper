# Implementation Plan

## Purpose

Define the concrete steps, risks, and validation methods for executing an engineering task.

## When to Use

- planning a feature
- preparing a refactor
- fixing a complex bug

## Deliverable Structure

### Existing Context

Summarize the current implementation.

### Proposed Changes

High-level description of what will be modified.

### Delegation Strategy (Optional)

If the implementation is complex, identify how the work will be partitioned and assigned to subagents, including the required context boundaries.

### Steps

Provide a prioritized, step-by-step roadmap for implementation.

### Risks

Potential implementation risks.

### Validation

How success will be verified (e.g., tests, metrics, manual QA).
