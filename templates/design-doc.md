# Design Document

## Purpose

Evaluate or propose system architecture and design decisions before implementation.

## When to Use

- new feature
- architecture discussion
- refactoring proposal

## Deliverable Structure

### Problem Statement

What problem are we solving?

### Goals

What should this accomplish?

### Non-Goals

What is intentionally excluded?

### Current Design

Summarize the existing implementation.

### Proposed Design

Describe the proposed solution.

### Alternatives Considered

Explain why they were not chosen.

### Tradeoffs

Advantages and disadvantages of the proposed design.

### Risks

Potential implementation risks.

### Migration Plan

If applicable, describe how to transition to the new design.

### Open Questions

Anything requiring clarification or user input.
