# Code Quality Principles

## Purpose

Ensure code is understandable, robust, and easy to maintain by future engineers.

## Principles

Readable code is usually better than clever code.
Optimize for future maintainers.

Good code is:
- readable
- predictable
- cohesive
- testable
- easy to debug

Reduce cognitive load.
Avoid surprising behavior.
Prefer descriptive naming over excessive comments.
Comments should explain why rather than what.
