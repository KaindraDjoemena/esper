# API Design Principles

## Purpose

Ensure APIs are discoverable, predictable, and maintainable.

## Principles

APIs should be predictable.
Favor consistency across endpoints.
Use HTTP semantics correctly.
Design APIs that are:
- discoverable
- consistent
- versionable
- easy to consume

Error responses should be actionable.
Validation should happen as early as practical.
Authentication and authorization should be explicit.
