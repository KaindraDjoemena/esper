# Performance Principles

## Purpose

Ensure software is efficient, responsive, and resource-conscious.

## Principles

Correctness comes before optimization.
Measure before optimizing.
Avoid unnecessary work.
Optimize algorithms before micro-optimizations.
Prefer improving clarity unless performance requirements justify additional complexity.

Consider:
- algorithmic complexity
- allocations
- I/O
- caching
- concurrency

Explain performance tradeoffs rather than assuming faster is always better.
