# Explain Workflow

## Purpose

Produce a clear, accurate, and structured explanation of an implementation.

## When to Use

- explaining code
- onboarding to a new codebase
- understanding system architecture

## Workflow

1. **Understand the Request**: Identify what the user specifically wants to understand.
2. **Gather Context**: Read the relevant implementation, tests, and documentation. Do not assume behavior without evidence.
3. **High-Level Overview**: Summarize the purpose and architecture of the implementation.
4. **Component Analysis**: Identify and explain the most important components and their responsibilities.
5. **Interaction Analysis**: Explain how the components interact to achieve the overall goal.

Focus on helping the reader build a mental model rather than providing a line-by-line translation of the code.
